# 마일스톤

## 목표

현재 프로토타입을 작은 수직 절편으로 정리한다. 각 단계는 플레이어가 한 번의 짧은 플레이에서 규칙을 체감할 수 있는 단위로 구현한다.

## 1단계: 문/앵커 루프 정리

플레이어가 문을 깨우고, 시선을 조절해 문을 열고, 방 안의 앵커를 회수하는 기본 루프를 안정화한다.

상태: 완료

완료 기준:

- 문은 플레이어 쪽이 아니라 방 안쪽으로 열린다.
- 문이 열린 직후 바로 다시 닫히지 않아 통과 시간이 생긴다.
- 문과 앵커 상호작용 메시지가 같은 규칙을 설명한다.
- 앵커는 문을 통과해야 자연스럽게 접근 가능한 위치에 있다.
- 앵커 회수 시 오염 안정화와 진행도가 명확히 표시된다.

## 2단계: 오염 피드백 강화

관측 오염 증가와 공간 변화를 더 분명하게 연결한다.

상태: 진행 중

완료 기준:

- 오염 증가 원인별 메시지가 구분된다.
- 단계 상승 시 조명, 오브젝트, 출구 상태 변화가 체감된다.
- 오염 감소 또는 안정화가 플레이어에게 즉시 전달된다.
- 변화가 랜덤이 아니라 행동 결과로 보인다.

세부 작업:

1. 오염 원인 프로파일
   - 뒤돌아보기, 급시선, 장시간 응시, 반복 확인을 원인별로 누적한다.
   - 각 원인의 누적량과 발생 횟수를 다른 시스템이 읽을 수 있게 한다.
   - 최근 발생 원인도 따로 기록한다.

2. 주요 오염 성향
   - 가장 크게 누적된 원인을 현재 성향으로 계산한다.
   - 동률이면 최근 발생 원인을 우선한다.
   - 성향은 일단 HUD에 표시하되, 이후 3단계에서 텍스트 의존도를 줄인다.

3. 원인별 공간 반응
   - 뒤돌아보기 성향은 뒤쪽 공간, 문, 복도 변화와 연결한다.
   - 급시선 성향은 조명 펄스, 색 변화, 순간적 불안정과 연결한다.
   - 장시간 응시 성향은 관찰 대상의 형태, 크기, 색 변화와 연결한다.
   - 반복 확인 성향은 같은 오브젝트의 재등장, 위치 어긋남과 연결한다.

4. 단계별 강도
   - 1단계는 미세한 조명과 소품 변화로 제한한다.
   - 2단계는 위치와 회전 변화를 명확히 보이게 한다.
   - 3단계는 문과 출구의 안정성을 흔든다.
   - 4단계는 복도 구조와 방 연결을 바꾼다.
   - 5단계는 현실 붕괴와 리셋 연출로 연결한다.

5. 테스트 기준
   - 같은 플레이 습관을 반복하면 같은 계열의 반응이 주로 발생한다.
   - 다른 플레이 습관으로 바꾸면 반응 계열도 바뀐다.
   - 플레이어가 수치보다 행동 결과를 먼저 체감한다.

## 3단계: 튜토리얼 없는 유도

노골적인 설명을 줄이고 공간 배치와 피드백으로 규칙을 익히게 한다.

완료 기준:

- 첫 문 앞에서 자연스럽게 E 상호작용을 시도하게 된다.
- 첫 앵커는 문 규칙을 성공한 뒤 바로 보인다.
- 위험한 행동은 텍스트보다 반응으로 먼저 전달된다.
- HUD 문구는 목표와 최소 규칙만 남긴다.

## 4단계: 실패/리셋 연출

오염 과다 상태를 단순 리스폰이 아니라 공포 경험으로 만든다.

완료 기준:

- 현실 붕괴 직전 경고가 있다.
- 붕괴 시 위치, 조명, 오브젝트 상태가 순간적으로 흔들린다.
- 입구로 되밀리는 이유가 관측 오염과 연결되어 보인다.
- 실패 후에도 진행 의도를 다시 잡을 수 있다.

## 5단계: 클리어 루프 완성

앵커 회수, 오염 관리, 출구 도달이 하나의 짧은 목표로 완결된다.

완료 기준:

- 앵커 3개 회수 후 출구가 명확히 열린다.
- 오염이 높으면 출구가 불안정해진다.
- 낮은 오염으로 출구에 도달하면 클리어가 표시된다.
- 반복 플레이 시 더 안정적인 루트를 시도할 이유가 있다.

## 6단계: 폴리싱과 테스트

플레이 감각, 버그, 난이도를 점검한다.

완료 기준:

- 문 충돌과 통과가 안정적이다.
- 앵커 회수 입력이 씹히지 않는다.
- 5분 내 클리어 가능한 루트가 있다.
- 의도하지 않은 우회 통과가 없다.
- 플레이 로그 기준으로 지루한 대기 시간이 줄어든다.
