# 개발 계획

## 목표

첫 개발 목표는 전체 게임 완성이 아니라, 관측 오염이 실제 공포 경험으로 작동하는지 검증하는 Unity 프로토타입 제작이다.

## 권장 엔진

- Unity
- 1인칭 3D
- 작은 실내 맵
- 모듈형 복도/방 구조

## 구현 우선순위

### 1단계: 기본 이동과 공간

- 1인칭 이동
- 카메라 회전
- 작은 복도/방 구성
- 문 열기/닫기
- 기본 상호작용

### 2단계: 행동 추적

- 뒤돌아보기 감지
- 응시 시간 측정
- 같은 오브젝트 반복 확인 감지
- 카메라 급회전 감지
- 관측 오염 수치 증가

### 3단계: 관측 오염 반응

- 시야 밖 오브젝트 위치 변경
- 오래 본 오브젝트 형태 변경
- 문 위치 또는 연결 상태 변경
- 조명 미세 변화
- 변화 강도 단계화

### 4단계: 클리어 조건

- 관측 오염 기반 출구 활성화
- 오염 과다 시 출구 불안정화
- 안정 상태에서만 탈출 가능
- 클리어 후 행동과 오염 관계 암시

### 5단계: 연출 보강

- 지연 발소리
- 시야 밖 변화 사운드
- 미세한 구조 차이
- 목표 텍스트 변화
- 오염 과다 시 현실 붕괴 연출

## 프로토타입 핵심 시스템

### ObservationContamination

관측 관련 행동을 누적해 관측 오염 수치를 관리한다.

필요 기능:

- 오염 증가
- 오염 감소 또는 안정화
- 현재 단계 계산
- 오염 과다 상태 판단

### PlayerBehaviorTracker

플레이어의 불안 행동을 감지한다.

필요 기능:

- 뒤돌아보기 감지
- 급격한 시점 회전 감지
- 특정 대상 응시 시간 측정
- 같은 대상 반복 확인 횟수 기록

### ContaminationResponder

오염 단계에 따라 공간 변화를 실행한다.

필요 기능:

- 오브젝트 위치 변경
- 오브젝트 활성/비활성
- 조명 상태 변경
- 문 연결 상태 변경
- 출구 안정/불안정 전환

### ExitController

클리어 가능 상태를 관리한다.

필요 기능:

- 출구 활성 조건 확인
- 오염 과다 시 출구 이동 또는 비활성화
- 안정 상태에서 클리어 처리

## 첫 데모 맵 구성

- 시작 방
- 메인 복도
- 관찰 오브젝트가 있는 방 2개
- 막힌 출구
- 최종 출구

## 테스트 기준

- 플레이어가 뒤돌아보면 오염이 증가한다.
- 오염 증가 후 시야 밖 오브젝트가 변한다.
- 변형이 랜덤이 아니라 행동과 연결되어 보인다.
- 너무 자주 확인하면 출구 접근이 어려워진다.
- 확인 행동을 줄이면 클리어 가능하다.

## 주요 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 아무 일도 안 일어나는 게임처럼 보임 | 초반은 미세하게, 후반은 명확하게 변화 강도 조절 |
| 랜덤 이벤트처럼 느껴짐 | 모든 변화는 행동 로그와 오염 수치 기반으로 발생 |
| 퍼즐게임처럼만 느껴짐 | 정답 암기보다 행동 통제를 중심으로 설계 |
| 3D 구현 범위가 커짐 | 첫 데모는 작은 공간과 높은 변화 밀도에 집중 |
